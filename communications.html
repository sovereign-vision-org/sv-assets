Network Architecture

How a node
actually works.

Each node is a self-contained communications hub — and connects via three-tier backhaul to the central Guardian datacenter when available. Solar energy feeds a local microgrid. An edge compute cluster runs the mobile core, voice services, and civic applications. Four independent radio tiers broadcast simultaneously, and fall back gracefully when conditions demand it. If the datacenter is unreachable, the node continues Tier 1 and Tier 2 services locally without interruption.

Power Layer
Off-grid microgrid

6.4 kW rooftop solar array charges a 28–30 kWh lithium iron phosphate battery bank via dual charge controllers. An inverter-charger delivers conditioned AC throughout the shelter. Guardian monitors state-of-charge every 30 seconds and enforces three-tier load shedding when solar input is interrupted.

6.4 kW peak solar generation
6+ hrs critical autonomy at Tier 1 load
Compute Layer
Edge intelligence

Four industrial edge computers run the mobile LTE core, VoIP gateway, system logging, and civic applications respectively. A dedicated AI inference module handles vision, voice, and environmental analysis at the edge — no cloud dependency, no latency overhead, no foreign data exposure.

4 edge compute nodes
2× AI inference modules
Radio Layer
Four-tier signal

The mast carries three LTE antennas at 120° spacing for island-wide mobile coverage, a satellite dish for LEO uplink, a community mesh access point for public Wi-Fi, and a VHF/UHF antenna for civic emergency broadcast. Each tier is independently operable — losing one does not affect the others.

4 independent signal tiers
120° LTE coverage spacing
Governance Layer
Guardian — edge & datacenter

Guardian Edge runs locally on the node: monitoring power, temperature, signal, and battery every 30 seconds; triggering storm protocols; managing VLANs; and logging every decision. Guardian Core runs in the sovereign datacenter: full AI orchestration, Arbor voice services, telemetry analysis, and Mirror Vault synchronisation. The edge operates fully independently if the datacenter is unreachable.

60s edge heartbeat interval
100% locally auditable at node level

Network Architecture

Four tiers.
One sovereign backbone.

Each node runs four independent communication layers simultaneously. When one layer is degraded by weather, power loss, or hardware failure, the others carry the load — automatically, without human intervention.

01
Pule Loto · Private Core
Private Mobile Network

Island-owned mobile connectivity. SIM-based devices connect through a locally operated core that no external carrier controls. Includes civic voice-over-IP for island-wide calling. All traffic processed on-island — zero foreign telco dependency.

Always On · Tier 1 Critical
02
Ara Lagi · Sky Path
Satellite Uplink

Off-island internet access via low-earth-orbit satellite with automatic weather-adaptive realignment. Acts as the primary global link until the Vaka undersea cable reaches Tuvalu. Fibre uplink ports are provisioned and ready on every node.

Global Bridge · Tier 2
03
Kapua Loloa · Long Cloud
Community Mesh Network

Multi-band mesh coverage across homes, schools, clinics, and desalination nodes. Integrates with the ArborMesh grid for power-aware load balancing — automatically scaling bandwidth as battery levels drop. Public and visitor devices connect here.

Community Access · Tier 3
04
Leo Fakafotu · Voice of the Sky
Emergency Radio Repeater

Backup voice and emergency alert broadcasting. Guardian-triggered alerts auto-broadcast on storm alert. Radio-over-IP bridges let mobile users reach radio receivers island-wide through civic voice routing. Functions when all other tiers are down.

Emergency Fallback · Tier 4

Energy Architecture

From sunlight
to signal.

The full energy lifecycle of a node — from photon to data packet. Every watt is tracked, prioritised, and governed. The moment solar drops, Guardian automatically cascades load shedding to protect the critical core.

Guardian Power Protocol

Storm Mode.
Tiered load triage.

When battery drops below 35%, solar is lost, or Guardian issues a storm alert — the node automatically sheds non-essential load in priority order.

Once solar is restored and battery exceeds 60%, Tier 2 resumes gradually. Arbor displays current load status and allows stewards to veto or request manual override on any tier.

Critical Communications

Always On
  • Mobile Network Repeater + Relay
  • Guardian AI Edge Server
  • Network Firewall — Security & Routing
  • Satellite Uplink
  • Inter-Node Mesh Relay Link
  • Primary Cooling (throttled)
Guardian holds Tier 1 operational until battery reaches absolute minimum threshold — approximately 6+ hours of critical autonomy from a full charge.

Priority Services

Solar-Dependent
  • Fibre Uplink (if cable available)
  • Admin Mesh (responders only)
  • Emergency Radio Broadcast (push only)
  • Environmental Sensors (log only)
  • Interior Cameras (motion-triggered)
Tier 2 resumes once solar input is restored and battery state-of-charge exceeds 60%. Arbor notifies stewards before restoring each service.

Non-Essential Load

Cut First
  • Public Mesh Network
  • Secondary Cooling Unit
  • Community Projector Feed
  • Non-Critical Camera Feeds
  • Ambient LED Lighting
Tier 3 remains offline until battery is fully recovered and Tier 2 has been stable for 30 minutes. Stewards may manually override through Arbor.

Edge Degradation Mode

Datacenter Offline
  • Mobile LTE Core — continues locally on edge node
  • Satellite Uplink — active, reroutes to backup path
  • Guardian Edge — local monitoring and storm protocol
  • Civic VoIP — continues via local FreeSWITCH gateway
  • Arbor Interface — limited mode, no AI inference
  • Sensor telemetry — buffered locally, syncs on reconnect
  • Guardian Core AI services — unavailable until datacenter restored
  • Mirror Vault sync — queued, resumes automatically
Edge Degradation Mode is not a failure state — it is a designed operational mode. Nodes were built to sustain core communications services indefinitely without the datacenter. When the datacenter reconnects, buffered telemetry syncs automatically and full Guardian Core services resume without manual intervention.
Network Sovereignty

Ten isolated
network zones.

Sovereignty is not just about who owns the hardware. It is about who controls the traffic. Every device on the network is assigned to a dedicated VLAN with enforced isolation — citizens never share a data path with infrastructure systems.

VLANZoneDevices & SystemsPriorityKey Rule
1 Management Switches, firewalls, servers, Guardian terminals High MAC-locked · port-controlled
20 Edge Compute Edge servers, AI inference modules, Guardian Edge High Guardian AI coordination
30 LTE Core Mobile core server, SIM routing, FreeSWITCH Med–High NAT policies enforced
40 Satellite Satellite terminal, WAN router uplink High Isolated WAN interface
50 Fibre Uplink SFP ports, backbone switch, router uplink High Enables Vaka cable peering
60 Public Wi-Fi Mesh APs, phones, tablets, visitor devices Low Rate-limited · Guardian-capped
70 Surveillance Cameras, access readers, NVR streams Medium VLAN tagging on PoE ports
80 IoT / Environmental PM2.5 sensors, rain gauge, CO₂, ultrasonic units Medium Guardian logs & alerts only
90 Guardian Civic Arbor terminals, projector, flow controller Medium Edge control for kiosks
99 Quarantine Unknown or unapproved devices pending Guardian review None Drops all external traffic

Guardian enforces VLAN policy continuously. Any device that drifts from its assigned zone triggers an alert and is automatically migrated to VLAN 99 pending steward review. No manual firewall management required.

AI Civic Systems
Guardian
& Arbor

The civic intelligence layer that monitors, manages, and communicates — in Tuvaluan.

On-Island · Open-Source · Auditable
"The network does not belong to a company. It belongs to the people — and Guardian is their steward."
Real-time node monitoring. Guardian polls every comms node continuously — detecting hardware faults, signal degradation, power anomalies, and temperature spikes before they cause outages. Predictive alerts trigger before failure, not after.
Storm protocol automation. On cyclone alert, Guardian automatically retracts the mast, switches to storm power mode, sheds Tier 3 load, and broadcasts an emergency alert via the civic radio repeater — no human action required.
VLAN isolation and firewall logic. Guardian manages network segmentation — separating the mobile core, satellite uplink, public mesh, IoT sensors, civic kiosks, and surveillance into isolated VLANs with enforced traffic policies. No cross-contamination.
Firmware and update sovereignty. Over-the-air firmware updates deploy with rollback safeguards. All updates are staged locally from an on-island mirror vault — no dependency on external servers or internet connectivity during the update process.
Full audit trail. Every automated decision Guardian makes — load shed, storm trigger, firmware push, VLAN change — is logged locally as a timestamped record, readable by the community council at any time.
Mirror Vault continuity. All audit logs, configuration snapshots, AI model states, and system telemetry replicate continuously to geographically distributed Guardian Mirror Vaults — including future oil-rig archive locations. Sovereignty is preserved even if the primary datacenter is destroyed.
Sovereign datacenter backbone. Full Guardian intelligence — Arbor voice AI, telemetry processing, predictive analytics, MQTT message bus, and knowledge retrieval — runs on a sovereign GPU compute cluster. No foreign cloud. No subscription dependency. All AI services are co-located with the network they govern. Explore the datacenter →
Arbor — The Civic Interface

Arbor translates Guardian's data into plain-language guidance for stewards — in Tuvaluan. Step-by-step maintenance instructions, live system status, storm alerts, and load override controls are all accessible through Arbor terminals installed at each civic node.

Deployment Roadmap

Phased across
the archipelago.

01
Phase 1 · Funafuti Core
10 Nodes — Civic & Critical Sites

Deploy 3 nodes across Funafuti — covering Vai Tapu desalination, the school cluster, and the main civic hub. Establish private mobile core and Guardian integration. Launch steward training and youth workforce certification programme.

02
Phase 2 · Outer Islands
Archipelago-Wide Expansion

Extend the mesh to outer islands and maritime docks. Each island cluster receives a satellite-enabled node for off-island uplink. Youth stewards certified in Phase 1 assume primary inspection and maintenance roles, guided by Arbor in Tuvaluan.

03
Phase 3 · Fibre Integration
Undersea Cable Connection

Seamless integration with the Vaka undersea cable network. Pre-provisioned fibre ports on existing nodes activate with no hardware replacement required. The satellite tier transitions to a resilience-redundancy role only.
